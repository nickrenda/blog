treemac | A command-line tool for creating file trees on OS X

If you use the Linux or Windows operating systems you may be aware that each has its own command-line tool to draw a file tree. However, this tool is missing by default from the OS X Terminal. That was the motivation behind this project.

Download

To get started with treemac, just download the executable binary here . Make sure you have execution rights by entering the command chmod 755 tree. Then just mv the binary to your /usr/local/bin/ folder or anywhere else you want to put it (just make sure that it's in your $PATH or that you create an alias).

Compile from Source

If for some reason the binary executable file does not run on your machine, you can follow these instructions to download and compile the program from source.

Note that this software is written in C++ and depends on the Boost library. If you don't have Boost already, I recommend installing the latest version via homebrew. Simply enter brew install boost. By default, this should install the Boost header files in /usr/local/Cellar/boost/1.*/include/ and the binaries in /usr/local/Cellar/boost/1.*/lib/. This is where I assume that you have them for the compilation step in the install procedure. If this is not the case, be sure to complete step 3 in the installation process (see below).

  1. cd to a directory where you would like to download the files.
  2. Clone the GitHub repository using git clone https://github.com/epwalsh/treemac.
  3. If the boost library is not installed in the location mentioned above, you will have to make a slight change to the Makefile. Open the Makefile and change the path on the line that says BOOST = /usr/local/Cellar/boost/1.* to the path of the boost library on your machine.
  4. Make sure you are in the root folder of the newly downloaded repository and type make followed by make install.

Usage

Once installed you can cd to any directory you want to create a file tree and just enter tree. You can also just add the path onto the command like you would with ls or cd. This will print a file tree that by default ignores all hidden files. If you want to show hidden files, use the -s option. You can also output the tree to a file using the standard redirection option like so: tree > ~/Desktop/myfiletree.txt. You could also use the [-o file] option.

Output the results to a file.

->> tree > ~/Desktop/tree.txt

Here I create a tree with hidden files in my ~/GitHub/ directory and save the result to a file on my desktop called github_tree.txt.

->> tree -s -o ~/Desktop/github_tree.txt ~/GitHub

To see all of the available options use the --help option, or -h for short.

->> tree --help
Usage: tree [-s] [-t theme] [-c color] [-o outfile] [directory]

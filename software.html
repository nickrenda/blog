viShiny | Interactive visualizations

Built using the Shiny framework for R, viShiny is an interactive web application that demonstrates certain statistical concepts. This is an ongoing project of mine and a tool that I use when teaching introductory statistics. viShiny implements a modular design (inspired by gammaram/intRo) that makes adding new topics quick and easy.

To use this app you have two choices. Either navigate to epwalsh.shinyapps.io/viShiny or run the app locally. To run locally, make sure you have R and the R package "shiny" installed, then from within R shiny::runGitHub("epwalsh/viShiny"). Similarly you could clone this repository and cd in the terminal to the root directory ~/.../viShiny. Then start R from within the same terminal window and enter

library(shiny)
runApp()

To report a bug or suggest a feature, submit an issue at github.com/epwalsh/viShiny/issues. Suggestions and contributions are welcome.

filetree | Easily create simple file trees using R

Have you ever wondered how to make a beautiful represenation of all the files within a directory? Not a command-line guru? No problem. Now you can make elegant file trees right from within R thanks to this neat little package complete with themes and the option to show hidden files. You can also export your file tree to a document.

Install

Assuming you have devtools installed,

devtools::install_github("epwalsh/filetree")

Examples

# Grow a file tree in the current working directory.
tree1 <- grow()

# Draw the file tree in the console.
draw(tree1)

treemac | A command-line tool for creating file trees on OS X

If you use the Linux or Windows operating systems you may be aware that each has its own command-line tool to draw a file tree. However, this tool is missing by default from the OS X Terminal. That was the motivation behind this project.

Before Install

Note that this program is written in C++ and depends on some boost header files. If you don't have boost already, I recommend installing the latest version via homebrew. Simply enter brew install boost. By default, this should install the boost header files in /usr/local/Cellar/boost/1.*/include/ and the binaries in /usr/local/Cellar/boost/1.*/lib/. This is where I assume that you have them for the compilation step in the install procedure. If this is not the case, be sure to complete step 3 in the installation process (see below).

Install

To install and use the treemac command-line tool you just have to follow a few short steps.

  1. cd to a directory where you would like to download the files.
  2. Clone the GitHub repository using git clone https://github.com/epwalsh/treemac.
  3. If the boost library is not installed in the location mentioned above, you will have to make a slight change to the Makefile. Open the Makefile and change the path on the line that says BOOST = /usr/local/Cellar/boost/1.* to the path of the boost library on your machine.
  4. Make sure you are in the root folder of the newly downloaded repository and type make. This will create two new folders: build/ and bin/. The bin/ folder contains the executable file called tree, which is the only file that you need now.
  5. Move the tree file to wherever you want to keep it on your Mac. For me, that is a folder I created in my home directory called ~/bin/.
  6. In your .bash_profile create an alias that points to the executable file. For me, I use alias tree="~/bin/tree".

Usage

After you source your .bash_profile you can cd to any directory you want to create a file tree and just enter tree. This will print a file tree that by default ignores all hidden files. If you want to show hidden files, use the -s option. You can also output the tree to a file using the standard redirection option like so: tree > ~/Desktop/myfiletree.txt.
